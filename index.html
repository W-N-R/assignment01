⚔️ Sizing Duel: content-box vs border-box

box-sizing: content-box

This is the default browser behavior. Width and height properties specify only the inner content box dimension. Any padding or borders are added to the outer dimensions.

  • Formula: Actual rendered size = Specified size + Padding + Border
  • Problem: Makes styling layouts unpredictable because adding padding or border breaks grid columns or custom widths!
  • Analogy: Buying a TV of width 50 inches, but when it arrives, the plastic frames (borders) and packaging add 5 extra inches on each side.

box-sizing: border-box

This is the industry standard/modern practice. The width and height properties specify the entire box size including content, padding, and border.

  • Formula: Actual rendered size = Specified size (Padding and border shrink the inner content box size dynamically)
  • Benefit: Makes designing extremely predictable. If you specify width: 300px, it will always render as exactly 300px wide.
  • Analogy: Buying a TV of width 50 inches, and the plastic frame is included inside that 50-inch measurement.

Margin

The space outside the border. It separates this element from adjacent page components. Margins do not have backgrounds (they are transparent) and do not respond to clicks.

Border

The perimeter around the padding. It serves as a visual separator. You can style its thickness, color, and design (solid, dashed, dotted, double, etc.).

Outline

A line drawn outside the border edge. Unlike borders, outlines do not take up space, do not affect box sizing calculations, and can overlap other elements.

Padding

The space inside the border. It protects the content from touching the border edges. It inherits the background of the element and increases click targets.

Content

The actual core content (text, image, nested divs) that you put inside the tags. Its size is controlled directly by the width and height properties.
